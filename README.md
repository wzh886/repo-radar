# repo-radar

A small engineering utility for screening GitHub repositories, tasks, and issue-based work before you spend real delivery time on them.

## Why it exists
Not every repository is worth your time. `repo-radar` is intended to help evaluate whether a repo or coding task is active, credible, and economically sensible.

## Planned capabilities
- summarize repo activity and freshness
- inspect issue clarity and likely acceptance path
- flag low-signal / low-reward tasks
- support paid-task triage and delivery planning
- provide a compact CLI for repository screening

## Example use cases
- deciding whether a GitHub task is worth taking
- screening repositories before making a fork
- comparing multiple paid tasks for ROI
- building a repeatable repo-evaluation workflow

## Status
Initial scaffold with simple repository scoring logic.

## Tech
- Python 3.11+
- standard library first
- built for scripting and extension
